## Entscheidungen

- **BE-/E2E-Tests:** Umgebung hat keinen lokalen Postgres/Docker → BE-Test-Suite
  nicht ausführbar. Gate dafür: `backend npm run type-check`. In jeder Aufgabe,
  die Backend berührt, vermerkt.
- **News-Ordner → `Post`-Modell (nicht `FeedPost`):** Die Dateinamen im Ordner
  `News/` (Forum_der_Kreislauf, Schutz_von_Meeresschildkroeten, LED) matchen
  exakt auf die drei `Post`-Zeilen (Projekt-Lifecycle-News), nicht auf die
  importierten griechischen FeedPosts (FB-Folders 1–20, unverwandte Titel). Das
  `Post`-Modell ist via feedController-Adapter Teil des öffentlichen Feeds, also
  erscheinen die Bilder dort. needsReview-/Übersetzungs-Pipeline der FeedPosts
  bleibt unangetastet.
- **C1 Breite vs. Lesbarkeit:** Laut Akzeptanzkriterium („Inhaltsbreite überall
  identisch zur Landing", „schmale max-w entfernen") wurden ALLE gelisteten Routen
  inkl. der Artikel-Detailseiten (News-/Lernen-/Ideen-Detail) auf 7xl gesetzt —
  ohne inneren Lesespalten-Constraint. Trade-off: Fließtext auf den Detailseiten
  läuft jetzt breit (>80 Zeichen/Zeile). Bewusst der expliziten Vorgabe gefolgt;
  falls Lesbarkeit Vorrang haben soll, wäre ein innerer `max-w-prose` auf NUR dem
  Artikel-Body die Nachbesserung. Rewards-Seite (mehrere sektionierte Container)
  bleibt C1-seitig unangetastet und wird in J3/J4 behandelt.
- **Logos & Favicon (A1/A2):** Werden in Supabase Storage hochgeladen (Pflicht
  laut Aufgabe, idempotent, URLs im Report) UND zusätzlich als lokale Assets ins
  Repo gelegt (public/ + src/assets), damit Build/Tests/Dev nicht von Prod-Storage
  abhängen. Frontend referenziert die lokalen Assets.
